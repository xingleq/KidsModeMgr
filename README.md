# Kids Mode Manager (儿童模式管理工具)

Windows 儿童电脑使用时间管理工具，限制连续使用时长，超时强制锁屏并执行休息策略。

## 功能特性

| 功能 | 说明 |
|------|------|
| 强制锁屏 | 达到时间上限后自动断开用户会话 |
| 强制休息 | 休息期内重新登录会被立即踢出 |
| 开机自启 | 安装即自动启动，无需手动干预 |
| 崩溃自恢复 | 服务被异常终止后 5 秒内自动重启 |
| 计时持久化 | 关机重启不会重置当日已用时间，跨天自动清零 |
| 防时间篡改 | 使用系统运行时间（GetTickCount64）判断休息期，修改系统时钟无效 |
| 注册表 ACL | 安装时自动锁定注册表，普通用户无法篡改配置 |
| 锁屏前预警 | 剩余 3 分钟和 30 秒时在用户桌面弹窗提醒，3 秒自动关闭 |
| 实时状态面板 | GUI 实时显示今日已用、距锁屏剩余、休息期倒计时 |
| 时间单位友好 | 参数以分钟为单位输入，旁边实时显示换算秒数 |
| 使用记录日志 | 每次锁屏事件写入日志，在 GUI 内直接查看最近 50 条 |

## 使用说明

### 1. 启动管理工具

双击 `kids_mode_mgr.exe`，程序自动申请管理员权限（必须允许，否则无法管理服务）。

### 2. 安装服务（首次使用）

点击 **安装服务**，程序自动完成：
1. 注册 Windows 服务并设为开机自动启动
2. 立即启动后台监控
3. 配置服务崩溃自动恢复策略
4. 锁定注册表 ACL（普通用户只读）

### 3. 参数设置

- **最大使用时间**：输入分钟数，旁边实时显示换算秒数
- **强制休息时间**：输入分钟数
- **开启强制休息**：勾选后超时锁屏并进入休息期

点击 **保存配置** 后，若服务正在运行，工具会询问是否立即重启服务生效。

### 4. 实时状态面板

打开管理工具后，中部面板每 3 秒自动刷新：
- 今日已用时间 / 上限
- 距下次锁屏剩余时间
- 休息期剩余倒计时（红色显示）

### 5. 锁屏前预警

孩子使用时间临近上限时，桌面会弹出系统提示：
- **剩余 3 分钟**：提醒准备保存作业
- **剩余 30 秒**：提醒立即保存

弹窗 **3 秒后自动关闭**，无需点击确认。

### 6. 使用记录

界面底部日志面板直接显示最近 50 条锁屏记录，点击 **刷新记录** 更新内容。

日志文件路径：`C:\ProgramData\KidsModeMgr\usage.log`

## 安全机制

| 攻击方式 | 对应防护 |
|----------|----------|
| 修改系统时间跳过休息期 | GetTickCount64 系统运行时间验证，不受系统时钟影响 |
| regedit 篡改配置或休息时间 | 注册表 ACL 限制普通用户只读 |
| 任务管理器杀死服务 | 服务失败恢复策略，5 秒内自动重启 |
| 关机重启清零计时 | 每分钟持久化当日已用时间，重启后恢复 |

> **重要**：以上防护均基于孩子使用**标准用户账户**。若孩子拥有管理员权限，上述保护均可被绕过。强烈建议为孩子创建标准用户账户。

## 注册表路径

`HKLM\SOFTWARE\KidsModeMgr`

| 键名 | 类型 | 说明 |
|------|------|------|
| MaxUsage | DWORD | 最大使用时间（秒） |
| MandatoryRest | DWORD | 强制休息时间（秒） |
| EnableRest | DWORD | 是否启用强制休息（0/1） |
| LastForceLockTime | REG_SZ | 上次锁屏 Unix 时间戳 |
| CurrentUsage | DWORD | 当前已用秒数（服务每秒写入，GUI 读取） |
| DailyUsage_YYYYMMDD | DWORD | 当日累计使用秒数（跨天自动失效） |

## 常见问题

**Q: 修改参数后没生效？**
A: 保存配置后工具会自动询问是否重启服务，确认即可。

**Q: 孩子绕过了限制？**
A: 检查孩子账户是否有管理员权限。标准用户账户是最有效的系统级隔离手段。

**Q: 日志文件在哪里？**
A: `C:\ProgramData\KidsModeMgr\usage.log`，也可在 GUI 底部日志面板直接查看。
